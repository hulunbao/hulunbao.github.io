解决访问github速度慢的问题

2019年7月17日
| GitHub
| 阅读

问题

今天上访问GitHub总是很慢，加载不出来。以为是网络问题，后来发现不是于是上网查找之后才知道在国内访问GitHub速度慢是有跟多原因的，但是主要的原因是因为GitHub的分发加速网络的域名遭到DNS污染。

由于GitHub的加速分发CDN域名assets-cdn.github.com遭到DNS污染，导致无法连接使用GitHub的加速分发服务器，才使得大陆访问速度很慢。

解决

通过修改host文件来解决,将域名直接指向IP地址绕过DNS解析。

获取域名的IP地址

访问 IPAddress.com 获取域名IP地址

140.82.113.4	github.com  
151.101.185.194 github.global.ssl.fastly.net

修改hosts文件

将上面得到的结果添加到系统的hosts文件

Linux下hosts文件路径

etc/hosts

Windows下hosts文件路径

C:\Windows\System32\drivers\etc

改完之后问题得到解决！
